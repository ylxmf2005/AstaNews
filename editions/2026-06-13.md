# 🛰️ AstaNews — 2026-06-13（星期六）

> 周六论文不多，主线落在“发布与 agent 评测”：MiniMax M3 把昨天的 MSA 稀疏注意力从论文做成今天可部署的开源权重，NVIDIA 联手 Artificial Analysis 立起首个 agentic 推理基准 AgentPerf/AA-AgentPerf，而 UC Berkeley 的 ALE 在真实专业软件里给各家旗舰 agent 泼了盆冷水——刷榜分数和真会干活之间，鸿沟正被摆上台面。

## 1. ⚡ [serving] NVIDIA 公布业界首个多厂商开放 agentic AI 推理基准 AA-AgentPerf

如果你在做 agent serving，最头疼的问题之一一直没有公认答案：一套推理系统在保证延迟的前提下，到底能同时撑住多少个 AI agent？Artificial Analysis 联合 NVIDIA 推出的 AA-AgentPerf，号称是首个回答这个问题的多厂商开放基准——它不再喂静态 prompt，而是回放真实的 agent 轨迹（带工具调用、上下文长度横跨 5K 到 131K token），填补了 agentic 推理负载此前没有标准量化口径的空白。

测试用 DeepSeek-V4-Pro 做被测模型，设了三档 SLO 输出速度——30、100、300 tok/s，对应从“能用”到“跟手”的不同体验要求。核心数字很扎眼：GB300 NVL72 单位兆瓦能跑 61.4K 个 agent、每张 GPU 撑 57.5 个，而 H200 对应只有 2.6K/MW、1.4/GPU，每兆瓦差出约 20 倍。请求序列平均长约 27K token，并刻意注入了 1 秒的中位 CPU 工具延迟——这一点很重要，因为真实 agent 大量时间卡在等工具返回，不建模这段就是在刷数字。

需要泼一句冷水：这 20x 是 GB300 对 H200 的跨代对比，里面含了硬件代际红利，不是纯软件优化挤出来的；而且基准由 NVIDIA 深度参与、首发用自家平台领跑，立场不中立。但即便打个折，把“并发 agent 数 / 兆瓦”确立为一个可量化、可横向比的口径，本身对 serving 选型就是实打实的进步——以后谈 agent 推理成本，终于有了一把公开的尺子。

🔗 [一手源](https://developer.nvidia.com/blog/nvidia-achieves-leading-agentic-coding-performance-on-first-agentic-ai-benchmark/)

## 2. 🧠 [model] MiniMax M3 开源权重上线 HF：428B/23B 激活原生多模态 MoE，1M 上下文，MSA 把解码加速 15×

昨天大家还在读 MiniMax 那篇 MSA 稀疏注意力的论文，今天权重就真落地到 Hugging Face 了。对想在自己机器上跑多模态 agent 的团队，这条值得停下来：MiniMax M3 是一个原生多模态的 MoE 模型——文本、图像、视频从训练第一步就混在一起，而不是事后拼接的视觉模块，总参数约 428B，每次推理只激活约 23B，上下文长达 1M token。

真正的看点是它把论文里的稀疏注意力直接交付到了 serving。靠 MiniMax Sparse Attention（MSA，每个查询只挑最相关的若干块 KV 来算，而不是全量算），相对上一代 M2，在 1M 上下文下 prefill 提速 9×、decode 提速 15×，单 token 的计算量压到原来的 1/20。模型给了 thinking / non-thinking 双模式，vLLM、SGLang、Transformers 第一天就能部署。技术报告挂在 arXiv:2606.13392。

放进格局里看，长上下文 agent 最烧的就是显存带宽和解码速度，谁能把这两项压下来，谁的本地 agent 才跑得起。M3 把“训练阶段验证过的稀疏注意力”直接做成可部署权重，是目前能本地跑的最强开源多模态 agent 权重之一。当然，9×/15× 是官方模型卡自报的提速数字，真实收益还得看各家在自己负载上的实测，但作为开源权重，它至少把门槛实实在在地降到了能复现的程度。

🔗 [一手源](https://huggingface.co/MiniMaxAI/MiniMax-M3)

## 3. 📊 [eval] UC Berkeley 放出 Agents' Last Exam（ALE）：在 NX/Unreal/AE 里真干活，GPT-5.5 反超 Fable 5

各家旗舰模型在 SWE-Bench、HLE 上动辄八九十分，但那真的等于“会干活”吗？UC Berkeley 放出的 Agents' Last Exam（ALE）给了一盆冷水：它不在沙箱里跑代码题，而是让 agent 进 Siemens NX 建 3D 模型、在 Unreal 里搭场景、用 After Effects 做合成——真实专业软件里的真实任务。结果是最难一档，Fable 5 和 GPT-5.5 全部 0 分。

更耐人寻味的是排名翻转。ALE 总通过率榜上，GPT-5.5+Codex 以 24.0% 夺冠，GPT-5.5+ALE Claw 23.0% 第二，Fable 5+Claude Code 只有 22.0% 排第三，Opus 4.7 拿 18.4%，Opus 4.8 垫底 15.8%。这和大家熟悉的格局几乎反过来——在 SWE-Bench Pro 上 Fable 系是 80.3% 对 GPT 的 58.6%，HLE 上 64.5% 对 52.2%，都是碾压；一换到真实工具链，碾压就消失甚至反转了。

提出团队不是无名之辈，正是 MMLU、MATH、CyberGym 的作者，做基准是老本行。需要留意的是这组数字目前来自量子位对 UC Berkeley 团队的转述，原始榜单细节值得回查；但方向性结论很清楚：现有刷榜分数和“在专业软件里真正完成一个项目”之间，存在一道还没被跨过的鸿沟。对所有把 agent 能力当卖点的人，这是最直接的冷水。

🔗 [一手源](https://www.qbitai.com/2026/06/434774.html)

## 4. 🏗️ [infra] NVIDIA Blackwell Leads on First Agentic AI Infrastructure Benchmark (AgentPerf)

如果你负责 infra 选型，传统那种“单次 LLM 调用能跑多少 token/s”的基准其实越来越不够用了——agent 工作负载是多步轨迹、夹着工具调用延迟、上下文一路膨胀，跟单次调用完全两码事。Artificial Analysis 推出的 AgentPerf 就是冲这个缺口来的，自称业界首个面向 agentic 工作负载的推理基准，轨迹取自真实编码 agent。

首发结果里 Blackwell 平台领跑。给出的关键数字是：在 DeepSeek V4 Pro 工作负载、20 与 60 tokens/s 两档 SLO 下，GB300 NVL72 单位兆瓦能运行的 agent 数最高达 HGX H200 系统的 20 倍。除了这个 20x 能效比，原文没有公开绝对吞吐或延迟数字，所以这更像是一张“能效比海报”而非完整跑分表。

这条和同日那篇 AA-AgentPerf 开发者博客是一回事的两个出口，互为印证。价值在于“为 agent 负载定义新基准类别 + 给出一手厂商数据”，对 serving/infra 选型有直接的方向性指导。但同样要记住：基准由 NVIDIA 参与、首发自家平台夺冠，20x 含跨代硬件红利，在第三方非 NVIDIA 平台复跑之前，把它当上限参考、别当结论。

🔗 [一手源](https://blogs.nvidia.com/blog/nvidia-blackwell-agentperf-artificial-analysis/)

## 5. 🤖 [agent] 把用户纠正编译成运行时强制约束:TRACE 让编码 agent 真正记住偏好

用过编码 agent 的人都遇到过这种憋屈：你纠正它一次（“别用 print 调试”“缩进用 4 空格”），它当场改了，下一会话又犯。研究者把这个问题量化了出来——即便接了偏好记忆，在 Mem0 方案下仍有 57.5% 的偏好检查照样被违反。问题不在“没记住”，而在“记住了也只是建议，能被悄悄跳过”。

TRACE 这篇论文改的是偏好的表示方式。它把用户聊天里的纠正挖出来，重写成一条条原子规则，再编译成任务完成前必须通过的运行时检查——于是“记住的偏好”从可选建议变成了硬性执行约束，agent 不满足就完不成任务。这个思路的关键转变，是从“让模型记得更牢”转向“在执行环节强制兑现”。

效果相当显著：在 ClawArena 的留出集上，偏好违反率从 100% 降到了分布外（OOD）任务的 2.0%、分布内（ID）任务的 37.6%；作为对照，Mem0 记忆仍留下 57.5% 的检查未通过。ID 任务还有 37.6% 没压住，说明运行时约束也不是万能解。但把偏好从“软记忆”升级为“硬门禁”这个方向，对正经做长期协作的编码 agent 很有启发——记得住，更要管得住。

🔗 [一手源](https://huggingface.co/papers/2606.13174) · [讨论](https://arxiv.org/abs/2606.13174)

## 6. 🤖 [agent] 异构 agent 之间直接传 KV-cache:跨模型的读心术对齐

多 agent 系统里，不同模型之间几乎都靠“说话”沟通——一个 agent 把想法解码成文本，另一个再编码读进去。这一来一回既有信息损耗，又费算力。一个更诱人的替代是直接传 KV-cache（模型内部的注意力缓存，相当于“它脑子里正在想的东西”），但难点在于：两个不同的模型，内部表示空间根本对不上。

这项工作攻的正是异构模型间的隐空间对齐。做法是用一个轻量的跨模型 cache 变换，配上两阶段训练——先学重建（保证信息不丢），再学生成（保证能接着往下用），让一个 agent 把“看到了什么”和“怎么想的”一并传给另一个不同架构的模型。它的信息结构分析还揭示了一个有意思的对偶：上下文感知的传输靠稀疏的推理信号，上下文无关的传输靠稠密的语境保留——两类信息走的是不同的路子。

实验上，在 Qwen3-4B/8B/14B 之间全部 6 个传输方向、6 个域内外基准上，都超过了此前的异构基线。需要客观看：实验目前只覆盖 Qwen3 同系列的不同尺寸，跨厂商、跨架构（比如 Qwen 传给 Llama）能否成立还未验证，离“任意模型间读心术”尚远。但对多 agent 协作这条线，把通信从有损的文本换成隐空间直传，是一个值得盯的方向。
